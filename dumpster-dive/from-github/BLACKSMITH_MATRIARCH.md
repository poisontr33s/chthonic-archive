## The Forge Protocol

Every file entering `dumpster-dive/` passes through Sister Ferrum's assessment:

```
RECEIVE → ASSESS → HEAT → HAMMER → QUENCH → TEMPER → SLAG
```

### Phase 1: RECEIVE
- File enters `dumpster-dive/` root or appropriate subdirectory
- Initial triage: Is this code, config, documentation, or conceptual fragment?

### Phase 2: ASSESS
- **Ore Rating (1-5):**
  - **⚗️ 1**: Pure slag. No recoverable value. Document as anti-pattern.
  - **⚗️ 2**: Low-grade ore. Fragments of useful logic.
  - **⚗️ 3**: Medium ore. Functional but needs refactoring.
  - **⚗️ 4**: High-grade ore. Minor cleanup required.
  - **⚗️ 5**: Refined metal. Ready for integration with validation.

### Phase 3: HEAT
- Bring ore into analysis context
- Apply relevant lenses (architectural, semantic, temporal)
- Identify structural impurities (hardcoded values, deprecated APIs, inconsistent patterns)

### Phase 4: HAMMER
- Strike to separate valuable from slag
- Refactor, extract, or document recoverable patterns
- Create integration-ready artifacts

### Phase 5: QUENCH
- Apply FA⁴ (Architectonic Integrity) validation
- Verify logical soundness, conceptual coherence, structural consistency
- If validation fails: return to HEAT phase or move to SLAG

### Phase 6: TEMPER
- Final integration preparation
- Document origin, transformations applied, integration point
- Update ORE_MANIFEST.json with final rating and notes

### Phase 7: SLAG
- Artifacts that cannot be salvaged
- Preserve as pedagogical material (what NOT to do)
- May be revisited if context shifts
